Angular Project Files & Explanations

1. angular.json
   The CLI configuration file for the workspace, specifying project settings, build options, assets, styles, scripts, environments, and configuration targets (like dev/prod builds).

2. tsconfig.json
   The root TypeScript configuration file for the workspace, defining compile options, base path, and target settings inherited by specific projects.

3. tsconfig.app.json
   The application-specific TypeScript configuration that inherits from the root tsconfig.json, tailoring settings for compiling the browser application.

4. package.json
   The npm package manifest containing project dependencies, devDependencies, metadata (like name/version), and scripts (e.g., build, test, start).

5. src/main.ts
   The main entry point of the application, responsible for bootstrapping the root module or component (e.g., bootstrapping the application with AppConfig).

6. src/app/app.config.ts
   The configuration file for standalone applications introduced in modern Angular, providing core providers (like routing, hydration, and other services) without needing an NgModule.

7. src/app/app.ts (corresponds to app.component.ts in standard layout)
   The root component of the application which serves as the base shell containing the selector, template, styles, and logic for the top-level view.

8. src/index.html
   The main HTML file served by the browser that acts as the entry page shell, containing the standard HTML structure and the custom root tag (e.g., <app-root>).

---------------------------------------------------------
Production Budgets Configuration (angular.json)
Under `architect > build > configurations > production`:
- "type": "initial" (applies to the initial bundle files loaded on app startup)
- "maximumWarning": "500kB" - Issues a build warning if the initial load bundle exceeds 500kB.
- "maximumError": "1MB" - Fails the build with an error if the initial load bundle exceeds 1MB.
These budgets control the compiled bundle sizes to ensure performance, fast page load speeds, and to prevent developers from accidentally introducing bloated dependencies into the application shell.

---------------------------------------------------------
Binding Types Difference
- One-way Property Binding ([property]="expression"):
  Data flows one-way from the component class to the DOM. When the component's state changes, the DOM updates. However, changes in the DOM (e.g., user inputs) do not update the component property automatically.
- Two-way Data Binding ([(ngModel)]="property"):
  Data flows bi-directionally between the component class and the DOM. Any state change in the component updates the DOM element, and any user input/change in the DOM updates the component property immediately (requires importing FormsModule).

---------------------------------------------------------
Compiled Application Files (dist/ folder)
Running `ng build` generates output in `dist/student-course-portal/browser/`.
The compiled application code (including components, routing, and business logic) is contained in the main bundle file:
`main-*.js` (e.g., `main-OZ2T5IEA.js` in the initial build).
Other generated files:
- `polyfills-*.js`: Polyfills for browser compatibility.
- `styles-*.css`: Compiled styles from the global styles.css and component styles.
- `index.html`: The HTML file with injected scripts and styles.
