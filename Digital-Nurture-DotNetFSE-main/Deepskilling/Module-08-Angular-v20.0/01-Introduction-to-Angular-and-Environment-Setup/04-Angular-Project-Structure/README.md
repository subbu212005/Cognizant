# Angular Project Structure

## Overview

When an Angular project is created using Angular CLI (`ng new AngularDemo`), it automatically generates a well-organized project structure. Each file and folder has a specific purpose that helps developers build, maintain, and scale Angular applications efficiently.

Understanding the project structure is essential before developing Angular applications.

# Explanation of Important Folders

## 1. `.angular/`

Contains Angular CLI cache files used to improve build performance.

---

## 2. `.vscode/`

Contains Visual Studio Code configuration files.

Examples:

* launch.json
* tasks.json
* extensions.json

---

## 3. `node_modules/`

Contains all installed npm packages and Angular dependencies.

**Note:** This folder is created automatically after running:

```bash
npm install
```

Do **not** edit files inside this folder manually.

---

## 4. `public/`

Stores static files such as:

* favicon.ico
* images
* icons

These files are served directly by the application.

---

## 5. `src/`

The **src** folder contains the application's source code.

It is the most important folder in an Angular project.

---

## app.ts

Contains the main application component logic.

Example:

* Variables
* Methods
* Component behavior

---

## app.html

Contains the HTML template displayed to users.

---

## app.css

Contains styles for the application component.

---

## app.spec.ts

Contains unit test cases written using Jasmine and Karma.

---

## app.config.ts

Stores application-wide configuration, such as providers.

---

## app.routes.ts

Defines application routes for navigation between pages.

---

## index.html

The main HTML page loaded by the browser.

It contains the root Angular element where the application is rendered.

---

## main.ts

The entry point of the Angular application.

It bootstraps the root application component.

---

## styles.css

Contains global CSS styles that apply to the entire application.

---

# Configuration Files

## angular.json

Main Angular CLI configuration file.

It defines:

* Build options
* Assets
* Styles
* Scripts
* Project configuration

---

## package.json

Contains:

* Project metadata
* Dependencies
* npm scripts

Example scripts:

```json
"start": "ng serve",
"build": "ng build",
"test": "ng test"
```

---

## package-lock.json

Locks the exact versions of installed npm packages to ensure consistent installations.

---

## tsconfig.json

Main TypeScript configuration file.

Defines compiler options and project settings.

---

## tsconfig.app.json

TypeScript configuration specific to the Angular application.

---

## tsconfig.spec.json

TypeScript configuration used for unit testing.

---

## .editorconfig

Defines consistent coding styles across different editors.

---

## .gitignore

Lists files and folders that Git should ignore.

Examples:

* node_modules/
* dist/
* cache files

---

## .prettierrc

Configuration file for the Prettier code formatter.

---

# Advantages of Angular Project Structure

* Organized and modular
* Easy to maintain
* Scalable for large projects
* Supports team collaboration
* Easy configuration management
* Built-in testing support

The Angular project structure generated by Angular CLI organizes application code, configuration, dependencies, and assets in a logical way. Understanding the purpose of each file and folder makes Angular development easier and helps developers build maintainable and scalable applications.
