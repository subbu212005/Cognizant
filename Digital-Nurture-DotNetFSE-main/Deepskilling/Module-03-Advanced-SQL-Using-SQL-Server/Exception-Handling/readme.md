# Exception Handling

## Introduction

Exceptions are runtime errors that occur during the execution of SQL statements. SQL Server provides exception handling mechanisms to detect, manage, and respond to errors without abruptly terminating program execution.

Exception handling improves database reliability, maintainability, and debugging.

---

## What are Exceptions?

An exception is an unexpected event or error that occurs during query execution.

Examples:

* Divide by zero
* Constraint violations
* Invalid object names
* Data conversion errors
* Permission errors

---

## Types of SQL Server Exceptions

### 1. System Defined Exceptions

These exceptions are automatically generated by SQL Server.

Examples:

* Divide by zero error
* Primary key violation
* Foreign key constraint violation
* Invalid column name

### 2. User Defined Exceptions

These exceptions are explicitly raised by developers using:

* THROW
* RAISERROR

Example:

```sql
THROW 50001, 'Invalid Student ID', 1;
```

---

## Retrieving Detailed Error Information

SQL Server provides built-in functions to retrieve error details.

| Function          | Description                |
| ----------------- | -------------------------- |
| ERROR_NUMBER()    | Returns the error number   |
| ERROR_MESSAGE()   | Returns the error message  |
| ERROR_LINE()      | Returns the line number    |
| ERROR_PROCEDURE() | Returns the procedure name |
| ERROR_SEVERITY()  | Returns severity level     |
| ERROR_STATE()     | Returns state information  |

Example:

```sql
SELECT
    ERROR_NUMBER(),
    ERROR_MESSAGE();
```

---

## TRY...CATCH

The TRY...CATCH block is used to handle runtime exceptions.

### Syntax

```sql
BEGIN TRY
    -- SQL Statements
END TRY

BEGIN CATCH
    -- Error Handling Code
END CATCH
```

### Example

```sql
BEGIN TRY
    SELECT 10 / 0;
END TRY

BEGIN CATCH
    SELECT ERROR_MESSAGE() AS ErrorMessage;
END CATCH;
```

---

## Nested TRY...CATCH

SQL Server supports nested exception handling.

### Example

```sql
BEGIN TRY

    BEGIN TRY
        SELECT 10 / 0;
    END TRY

    BEGIN CATCH
        PRINT 'Inner Catch Block';
    END CATCH

END TRY

BEGIN CATCH
    PRINT 'Outer Catch Block';
END CATCH
```

---

## THROW Statement

THROW is used to generate custom exceptions.

### Example

```sql
THROW 50001, 'Invalid Student Record', 1;
```

### Advantages

* Modern error handling method
* Preserves original error information
* Easy to use

---

## RAISERROR Statement

RAISERROR generates custom error messages.

### Example

```sql
RAISERROR
(
    'Marks cannot be negative.',
    16,
    1
);
```

### Advantages

* Customizable messages
* Supports severity levels
* Useful for validation
