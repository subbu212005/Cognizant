# Notes

## Chrome DevTools

Chrome DevTools is a browser debugging tool used to inspect and debug web applications.

---

## Main Features

### Elements

- Inspect HTML
- Edit HTML
- Edit CSS
- View DOM Tree

---

### Console

Used to:

- Print messages
- Display errors
- Execute JavaScript

Example:

```javascript
console.log("Hello Angular");
```

---

### Sources

Used for debugging.

Features:

- Breakpoints
- Call Stack
- Scope
- Watch Variables
- Step Over
- Step Into
- Step Out

---

### Network

Displays

- HTTP Requests
- API Calls
- Response Time
- Status Codes

---

### Performance

Used to analyze

- Rendering
- JavaScript execution
- Memory usage

---

## Source Maps

Source Maps connect compiled JavaScript with original TypeScript files.

Advantages:

- Easier debugging
- Readable code
- Better error tracking

---

## Breakpoints

A breakpoint pauses execution at a specific line.

Types:

- Line Breakpoint
- Conditional Breakpoint
- DOM Breakpoint
- XHR Breakpoint

---

## Call Stack

Displays the sequence of function calls leading to the current execution point.

---

## Scope

Displays

- Local Variables
- Global Variables
- Closure Variables

---

## Best Practices

- Remove unnecessary console.log statements.
- Use breakpoints instead of excessive logging.
- Check browser console regularly.
- Monitor Network requests.
- Debug using source maps.

---

## Conclusion

Chrome DevTools provides powerful debugging capabilities for Angular applications, helping developers quickly identify and resolve frontend issues.
