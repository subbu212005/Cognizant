# Module 09 – Application Debugging

## Overview

This module focuses on debugging Angular applications using widely adopted tools such as **Google Chrome DevTools** and **Visual Studio Code**. Debugging is an essential part of frontend development that helps developers identify, analyze, and resolve issues efficiently.

Throughout this module, learners gain practical experience in inspecting the Document Object Model (DOM), debugging TypeScript code using source maps, setting breakpoints, inspecting variables, monitoring application execution flow, and debugging Angular applications directly from Visual Studio Code.

---

## Learning Objectives

After completing this module, learners will be able to:

- Debug Angular applications using Chrome DevTools.
- Inspect HTML elements and CSS using the Elements panel.
- Understand the DOM tree and application structure.
- Use the Sources panel to debug JavaScript and TypeScript.
- Configure and use the Visual Studio Code debugger.
- Set breakpoints and inspect variables during execution.
- Use watches and the call stack to analyze application behavior.
- Debug Angular services and application state.
- Improve application reliability by applying systematic debugging techniques.

# Exercise 1

## 01-Debugging-Angular-with-Chrome-DevTools

### Objective

To learn how to inspect and debug Angular applications using Chrome DevTools.

### Activities Performed

- Created a standalone Angular application.
- Executed the application using `ng serve`.
- Opened Chrome DevTools using **F12**.
- Inspected HTML elements using the Elements panel.
- Modified CSS properties in real time.
- Used the Sources panel.
- Set breakpoints.
- Executed the application step-by-step.
- Inspected variables and scope.
- Verified TypeScript source maps.
