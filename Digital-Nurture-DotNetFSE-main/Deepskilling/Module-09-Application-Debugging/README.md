# Module 09 – Application Debugging

This module focuses on debugging Angular applications using widely adopted tools such as Chrome DevTools and Visual Studio Code. It covers identifying, analyzing, and resolving frontend issues by inspecting the DOM, debugging TypeScript code via source maps, and examining application state.

## Topics Covered
- **Chrome DevTools Debugging:** Opening DevTools, DOM tree inspection, Elements panel styling, and CSS modifications.
- **JavaScript & TypeScript Debugging:** Using the Sources panel, setting programmatic breakpoints (`debugger`), stepping through execution, monitoring scope variables, and using source maps.
- **Visual Studio Code Debugging:** Configuring the debugger (`launch.json`), launching Chrome debugger attached to Angular applications, inspecting local variables, watches, and the call stack.
- **State Management & Services:** Best practices in debugging services, inspecting state, and monitoring execution flow.
