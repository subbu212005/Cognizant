- Open Closed Principle (OCP)
- Liskov Substitution Principle (LSP)
- Interface Segregation Principle (ISP)

- Singleton Pattern
- Factory Method Pattern
- Builder Pattern

### Structural Design Patterns
- Adapter Pattern
- Decorator Pattern
- Proxy Pattern

### Behavioral Design Patterns
- Observer Pattern
- Strategy Pattern

### Architectural Patterns

- Dependency Injection (DI)

### Key Learnings
- Importance of clean code and maintainable software design.
- Application of SOLID principles in object-oriented programming.
- Understanding the purpose and implementation of common design patterns.
- Benefits of loose coupling and high cohesion.
- Building scalable and reusable software solutions using design patterns.
