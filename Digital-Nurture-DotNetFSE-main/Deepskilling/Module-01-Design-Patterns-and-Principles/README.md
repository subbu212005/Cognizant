# Module 01 - Design Patterns and Principles

## Overview

This module focuses on the fundamental software design principles and design patterns used to build scalable, maintainable, and reusable applications. The concepts learned in this module help developers write clean code, reduce complexity, and improve software architecture.

## Learning Objectives

* Understand the importance of SOLID principles in Object-Oriented Programming.
* Learn how to design flexible and maintainable software systems.
* Explore commonly used Design Patterns and their real-world applications.
* Apply design principles to improve code quality and reusability.

## Topics Covered

### SOLID Principles

* Single Responsibility Principle (SRP)
* Open/Closed Principle (OCP)
* Liskov Substitution Principle (LSP)
* Interface Segregation Principle (ISP)
* Dependency Inversion Principle (DIP)

### Creational Design Patterns

* Singleton Pattern
* Factory Method Pattern
* Builder Pattern

### Structural Design Patterns

* Adapter Pattern
* Decorator Pattern
* Proxy Pattern

### Behavioral Design Patterns

* Observer Pattern
* Strategy Pattern
* Command Pattern

### Architectural Concepts

* Model View Controller (MVC)
* Dependency Injection
