# Proxy Design Pattern

## Definition
The Proxy Pattern provides a placeholder or surrogate object to control access to another object.

## Why Proxy?

- Controls access to objects.
- Improves security.
- Supports lazy loading.
- Enhances performance.

## Real World Example

An ATM acts as a proxy between the user and the bank account.

## Program Output

```
Accessing Secure Resource
```

## Conclusion

The Proxy Pattern controls access to an object while providing additional functionality such as security and performance optimization.
