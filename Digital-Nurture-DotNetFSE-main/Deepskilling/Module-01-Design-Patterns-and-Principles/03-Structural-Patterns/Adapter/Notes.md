# Adapter Design Pattern

## Definition
The Adapter Pattern allows incompatible interfaces to work together by acting as a bridge between them.

## Why Adapter?

- Improves compatibility between classes.
- Promotes code reusability.
- Integrates legacy systems with new systems.

## Real World Example

A mobile charger adapter allows a device with one type of plug to connect to a different power socket.

## Program Output

```
Playing MP3 File
```

## Conclusion

The Adapter Pattern enables classes with incompatible interfaces to collaborate without modifying their source code.
