# Decorator Design Pattern

## Definition
The Decorator Pattern adds new functionality to an object dynamically without changing its structure.

## Why Decorator?

- Extends functionality without modifying existing code.
- Follows the Open Closed Principle.
- Improves flexibility.

## Real World Example

Adding cheese, vegetables, or sauces to a pizza without changing the original pizza.

## Program Output

```
Basic Coffee
Coffee with Milk
```

## Conclusion

The Decorator Pattern allows behavior to be added dynamically while keeping existing code unchanged.
