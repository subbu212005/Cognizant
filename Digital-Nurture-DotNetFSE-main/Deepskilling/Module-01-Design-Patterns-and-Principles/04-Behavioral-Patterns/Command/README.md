# Command Design Pattern

## Definition
The Command Pattern encapsulates a request as an object, allowing parameterization of clients with requests.

## Why Command?

- Decouples sender and receiver.
- Supports undo/redo operations.
- Improves flexibility.

## Real World Example

A TV remote control sends commands to the television without knowing its internal implementation.

## Program Output

```
Light Turned ON
```

## Conclusion

The Command Pattern encapsulates actions into objects, making systems more flexible and maintainable.
