# Dependency Injection

## Definition

Dependency Injection (DI) is a design pattern in which dependencies are provided to a class from outside rather than being created inside the class.

## Why Dependency Injection?

- Reduces coupling.
- Improves testability.
- Increases flexibility.
- Simplifies maintenance.

## Real World Example

A student management system may use different database services.

Instead of creating the database service inside the application, it is injected through an interface.

## Program Output

```
Student Data Retrieved Successfully
```

## Conclusion

Dependency Injection improves maintainability by making components independent and easier to replace.
