# Getting Started

## What is Automated Testing?

Automated Testing is the process of using software tools to execute test cases automatically and compare actual results with expected results.

## Benefits of Automated Testing

* Faster execution
* Improved accuracy
* Increased test coverage
* Reusable test scripts
* Early bug detection

## Types of Tests

* Unit Testing
* Integration Testing
* System Testing
* Acceptance Testing

## Test Pyramid

* Unit Tests (Base)
* Integration Tests (Middle)
* UI Tests (Top)

The Test Pyramid recommends having more unit tests and fewer UI tests.

## Popular Testing Frameworks

* NUnit
* xUnit
* MSTest
* JUnit

## Using NUnit in Visual Studio

NUnit is a unit testing framework for .NET applications. It provides attributes and assertions to create and execute tests.

## Test Driven Development (TDD)

TDD is a development approach where tests are written before the actual code.

### TDD Cycle

1. Write a Test
2. Run the Test (Fail)
3. Write Code
4. Run the Test (Pass)
5. Refactor Code

## Learning Outcome

Learned the fundamentals of automated testing, testing frameworks, and Test Driven Development.
